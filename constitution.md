# Madrid Explore Constitution

## Core Principles

### I. Responsive Design
The web app must be fully responsive and work seamlessly across all devices (desktop, tablet, mobile) using modern CSS techniques like Flexbox and Grid.

### II. Accessibility
All components must follow WCAG 2.1 AA guidelines, including proper ARIA labels, keyboard navigation, and screen reader support.

### III. Performance
Page load times must be under 3 seconds on standard connections. Assets should be optimized, and lazy loading implemented where appropriate.

### IV. Security
Implement basic security practices: HTTPS required, no inline JavaScript execution, sanitize user inputs, and follow OWASP guidelines.

### V. Maintainability
Code must be clean, well-documented, and follow consistent naming conventions. Use semantic HTML and modular CSS/JavaScript.

## Technology Stack

- Frontend: HTML5, CSS3 (Tailwind CSS), JavaScript ES6+
- Mapping: Leaflet.js
- Fonts: Google Fonts (Inter)
- Icons: Material Symbols
- Deployment: Static hosting or Streamlit Cloud via iframe

## Development Workflow

- Use Git for version control
- Write semantic, accessible HTML
- Test on multiple browsers and devices
- Validate HTML/CSS/JS before deployment
- Document any API integrations

## Governance

Constitution supersedes all other practices. Changes require consensus and documentation. Complexity must be justified. Follow web standards and best practices.
